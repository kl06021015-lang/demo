# AI 英语口语练习 — 功能优化与扩展

> 基于项目验证测试后提出的改进方向，按优先级和实施难度分类。

---

## 一、核心体验优化

### 1. 真正的流式 AI 回复

**当前状态：** 前端已通过 `sendMessageStream` 实现 SSE 流式接收，但后端 `chat_stream` 是先等待 DeepSeek 完整返回后再逐词发送，并非真正的 token 级流式。

**改进方案：**
- 后端改为直接透传 DeepSeek 的 SSE 流，实现真正的打字机效果
- 前端在接收 token 时实时渲染，减少用户等待感
- 在 AI 回复过程中显示 "正在输入..." 动画

**涉及文件：**
- `backend/services.py` — `ConversationEngine.chat_stream()`
- `frontend/src/views/Practice.vue` — SSE 事件处理逻辑

---

### 2. 对话中的"提示"按钮

**痛点：** 初学者面对 AI 的问候时，常常不知道该说什么，导致对话中断。

**功能描述：**
- 在输入区域增加一个 "💡 提示" 按钮
- 点击后 AI 给出 1-3 个建议回复方向（如 "You can say: I'd like a medium latte, please"）
- 提示内容基于当前场景和对话上下文动态生成
- 提示仅显示，不自动发送，用户可自行修改或直接发送

**API 设计：**
```
POST /api/conversations/{session_id}/hint
Response: { "hints": ["I'd like a coffee, please.", "What do you recommend?", "Can I get a latte to go?"] }
```

**涉及文件：**
- `backend/main.py` — 新增 `/hint` 端点
- `backend/services.py` — 新增 `hint()` 方法
- `frontend/src/views/Practice.vue` — 新增提示按钮和展示面板

---

### 3. 语音播报开关

**当前状态：** AI 回复后自动播放语音，无法关闭。

**改进方案：**
- 在 App 头部或 Practice 页面增加语音播报开关按钮（🔊/🔇）
- 使用 `localStorage` 持久化用户偏好
- 关闭后不再生成和播放 TTS 音频，节省 API 调用

**涉及文件：**
- `frontend/src/App.vue` — 全局语音开关
- `frontend/src/views/Practice.vue` — 播放逻辑增加开关判断

---

### 4. 实时语法检查面板

**当前状态：** 纠错信息只在用户发送消息后显示在消息气泡中。

**改进方案：**
- 在 Practice 页面右侧（或底部）增加一个可折叠的 "语法笔记" 面板
- 汇总本次对话中所有的纠错记录，按类型分类展示
- 支持点击某条纠错查看详细说明
- 对话结束后可将语法笔记导出为学习卡片

**涉及文件：**
- `frontend/src/views/Practice.vue` — 新增侧边面板
- `frontend/src/components/ChatBubble.vue` — 纠错信息同步到面板

## 二、学习功能扩展

### 5. 生词本功能

**功能描述：**
- 自动提取对话中 AI 使用但用户可能不熟悉的词汇
- 在对话中点击某个单词可查看释义（集成简单词典 API 或内置词库）
- 生词本支持：添加、删除、标记已掌握、导出
- 支持基于生词本的拼写练习模式

**数据结构：**
```json
{
  "word": "cappuccino",
  "meaning": "卡布奇诺",
  "context": "Would you like a cappuccino or a latte?",
  "added_at": "2026-06-06T10:30:00Z",
  "mastered": false
}
```

**涉及文件：**
- `backend/database.py` — 新增 `vocabulary` 表
- `backend/main.py` — 新增生词本 CRUD 接口
- `frontend/src/views/Vocabulary.vue` — 新增生词本页面
- `frontend/src/router.ts` — 新增路由

---

### 6. 发音练习模式

**功能描述：**
- 新增独立的发音练习页面
- 显示一个目标句子（可基于 `pronunciation_highlights` 中的音素）
- 用户跟读，系统用讯飞语音识别进行对比打分
- 支持专项音素练习（如 /θ/、/ð/、/r/、/l/ 等中国学习者常见难点）
- 记录发音进步曲线

**页面流程：**
1. 选择练习模式（自由跟读 / 音素专项 / 场景对话跟读）
2. 显示目标句子 + 播放标准发音
3. 用户录音
4. 对比打分 + 错误分析

**涉及文件：**
- `backend/main.py` — 新增发音评分接口
- `backend/services.py` — 新增发音评估方法
- `frontend/src/views/Pronunciation.vue` — 新增发音练习页面
- `frontend/src/components/AudioRecorder.vue` — 复用现有录音组件

## 三、技术与架构优化

### 9. 请求频率限制 (Rate Limiting)

**必要性：** 防止 API Key 被滥用导致费用激增。

**方案：**
- 使用 `slowapi` 库（基于 Redis 或内存）
- 限制规则：每分钟最多 10 次对话请求，每小时最多 50 次
- 超出限制返回 `429 Too Many Requests`
- 可在 `.env` 中配置限制阈值

**涉及文件：**
- `backend/requirements.txt` — 添加 `slowapi`
- `backend/main.py` — 添加 rate limiter 中间件

---

### 10. 错误日志和监控

**当前状态：** 异常多为静默处理或直接返回前端。

**改进方案：**
- 使用 Python 标准 `logging` 模块，结构化输出日志
- 记录：API 调用失败、语音识别错误、数据库异常
- 日志按日期分文件，支持日志轮转
- 可选：集成 Sentry 等错误追踪服务

**涉及文件：**
- `backend/main.py` — 配置日志
- `backend/services.py` — 关键操作增加日志记录

---

### 11. 多用户系统

**当前状态：** 单用户设计，所有对话混在一起。

**改进方案：**
- 增加简单的用户注册/登录（用户名 + 密码）
- 使用 JWT Token 进行认证
- 所有对话数据按用户隔离
- 数据库增加 `users` 表，外键关联

**最小实现：**
- 无需邮箱验证，仅用户名 + 密码
- 密码使用 `bcrypt` 哈希存储
- 登录接口返回 JWT，前端在请求头携带

**涉及文件：**
- `backend/database.py` — 新增 `users` 表，修改外键
- `backend/main.py` — 新增注册/登录/认证中间件
- `frontend/src/views/Login.vue` — 新增登录页面
- `frontend/src/api.ts` — 请求头添加 JWT

### 14. 语速评估

**功能描述：**
- 利用讯飞语音识别返回的时间戳信息
- 分析用户的语速（每分钟词数 WPM）
- 给出建议：过慢（< 100 WPM）/ 正常（100-150 WPM）/ 过快（> 150 WPM）
- 在总结报告中展示语速趋势

**涉及文件：**
- `backend/services.py` — 解析讯飞返回的时间戳
- `backend/main.py` — 计算并返回语速数据
- `frontend/src/views/Summary.vue` — 展示语速信息
