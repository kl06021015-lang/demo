# 语音录音持久化 — 需求分析与实现方案

## 一、需求概述

用户在进行英语口语练习时，语音输入的内容应当被持久化保存到服务端。用户可以在对话中、历史记录回放中随时听到自己的原始录音，即使刷新页面、关闭浏览器重新打开，录音也能正常播放。

### 核心用户故事

1. **对话中回听** — 用户说完一句话后，可以在自己的消息气泡旁点击播放按钮，听到刚才自己的录音
2. **刷新不丢失** — 刷新页面后重新进入对话，之前的语音消息仍然可以播放
3. **历史回放** — 在练习记录中查看已结束的对话时，每条用户语音消息都可以播放
4. **存储可靠** — 录音文件持久化在服务端磁盘，不依赖浏览器内存

## 二、现状分析

### 2.1 当前语音数据流

```
浏览器                         后端                           数据库
───────                       ──────                         ──────
AudioRecorder.vue
  │ 录制 WAV Blob
  │
  └─► POST /api/conversations/{id}/message
        FormData { audio: Blob, filename: "recording.wav" }
                                    │
                                    ├─► SpeechService.transcribe(audio_bytes)
                                    │     返回 text
                                    │
                                    └─► database.append_turn(session_id, {
                                          turn, user_text, corrected_text,
                                          ai_text, corrections_json,
                                          pronunciation_score, pronunciation_note
                                        })
                                    
                                    ⚠️ audio_bytes 用完即丢弃，不保存
```

### 2.2 当前数据表结构 (`turns` 表)

```sql
CREATE TABLE IF NOT EXISTS turns (
    id                  INTEGER PRIMARY KEY AUTOINCREMENT,
    session_id          TEXT NOT NULL,
    turn_number         INTEGER NOT NULL,
    user_text           TEXT NOT NULL DEFAULT '',
    corrected_text      TEXT NOT NULL DEFAULT '',
    ai_text             TEXT NOT NULL,
    corrections_json    TEXT NOT NULL DEFAULT '[]',
    pronunciation_score REAL,
    pronunciation_note  TEXT NOT NULL DEFAULT '',
    FOREIGN KEY (session_id) REFERENCES conversations(session_id)
);
```

**问题：** 没有 `audio_path` 字段，音频数据无处关联。

### 2.3 当前 API 响应结构

`GET /api/conversations/{session_id}` 返回的 `messages` 数组：

```json
{
  "turn": 1,
  "user_text": "I want a latte",
  "corrected_text": "",
  "ai_text": "Great choice! ...",
  "corrections": [],
  "pronunciation_score": { "overall": 7.5, "note": "" },
  "pronunciation_note": ""
}
```

**问题：** 没有 `audio_url` 字段。

### 2.4 当前前端组件

| 组件 | 当前能力 |
|------|----------|
| `AudioRecorder.vue` | 录制 WAV、发射 `audio-ready` 事件、不保留 Blob |
| `ChatBubble.vue` | 仅 AI 消息支持音频播放（TTS 语音），用户消息无播放按钮 |
| `Practice.vue` | 发送后丢弃 Blob，不存储 |
| `History.vue` | 可进入已结束对话的 Practice 页，但无语音 |

### 3.1 整体架构

```
                                     ┌──────────────────────┐
                                     │   backend/data/      │
                                     │   audio/             │
                                     │     {session_id}/    │
                                     │       turn_001.wav   │
                                     │       turn_002.wav   │
                                     │       ...            │
                                     └──────────┬───────────┘
                                                │
浏览器                   后端                    │           数据库
───────                 ──────                   │           ──────
                        POST /message            │
AudioRecorder           │ 接收 audio bytes ──────┤
  │                     │ 保存到磁盘 ────────────┘

       播放按钮          │   ← 返回 audio_url 字段
```

## 四、后端实现

### 4.1 数据库迁移 — `database.py`

在 `turns` 表中新增 `audio_path` 字段：

```python
# init_db() 中增加迁移逻辑
conn.execute("""
    ALTER TABLE turns ADD COLUMN audio_path TEXT NOT NULL DEFAULT ''
""")  # 使用 try/except 捕获列已存在的错误
```

`append_turn()` 函数增加 `audio_path` 参数：

```python
def append_turn(session_id: str, turn_data: dict) -> None:
    conn.execute("""
        INSERT INTO turns (session_id, turn_number, user_text, corrected_text,
                           ai_text, corrections_json, pronunciation_score,
                           pronunciation_note, audio_path)
        VALUES (?, ?, ?, ?, ?, ?, ?, ?, ?)
    """, (
        session_id,
        turn_data["turn"],
        turn_data.get("user_text", ""),
        turn_data.get("corrected_text", ""),
        turn_data.get("ai_text", ""),
        json.dumps(turn_data.get("corrections", []), ensure_ascii=False),
        json.dumps(score) if score else None,
        turn_data.get("pronunciation_note", ""),
        turn_data.get("audio_path", ""),
    ))
```

`get_conversation()` 读取时返回 `audio_url`：

```python

"audio_url": f"/api/audio/{session_id}/{t['turn_number']}.wav" if t["audio_path"] else None
```

### 4.2 音频文件存储 — `main.py`

#### 4.2.1 消息端点修改 (`POST /api/conversations/{session_id}/message`)

```python
# 在接收 audio 的分支中增加保存逻辑
if audio_file:
    audio_bytes = await audio_file.read()
    
    # 确定 turn_number
    turn_number = len(existing_turns) + 1
    
    # 保存到磁盘
    audio_dir = DATA_DIR / "audio" / session_id

    audio_path = audio_dir / f"turn_{turn_number:03d}.wav"
    audio_path.write_bytes(audio_bytes)
    
    # 传给 SpeechService
    text = await speech.transcribe(audio_bytes)
    
    # 存储时记录路径
    turn_data = {
        "turn": turn_number,
        "user_text": text,
        "audio_path": str(audio_path.relative_to(DATA_DIR)),
        ...
    }
```

#### 4.2.2 新增音频文件服务端点 (`GET /api/audio/{session_id}/{filename}`)

```python
@app.get("/api/audio/{session_id}/{filename}")
async def serve_audio(session_id: str, filename: str):
    """Serve a saved audio recording."""
    audio_path = DATA_DIR / "audio" / session_id / filename
    if not audio_path.exists():
        raise HTTPException(status_code=404, detail="Audio not found")
    return FileResponse(
        path=str(audio_path),
        media_type="audio/wav",
        headers={"Cache-Control": "public, max-age=31536000, immutable"}
    )
```

新增 prop 和方法：

```typescript
const props = defineProps<{
  // ... 现有 props ...
  audioUrl?: string  // ← 新增
}>()

const audioPlaying = ref(false)
const audioElement = ref<HTMLAudioElement | null>(null)

function playUserAudio() {
  if (!props.audioUrl || audioPlaying.value) return
  audioPlaying.value = true
  const audio = new Audio(props.audioUrl)
  audioElement.value = audio
  audio.play().catch(() => { audioPlaying.value = false })
  audio.onended = () => { audioPlaying.value = false }
}
```

## 六、边界情况与风险

### 6.1 存储空间

| 场景 | 估算 |
|------|------|
| 单次录音 | ~95 KB（16kHz 16bit mono WAV，5 秒） |
| 一次 10 轮对话 | ~950 KB |
| 100 次对话 | ~95 MB |

**应对方案：**
- 删除对话时同步清理音频文件
- 可选：在 Dashboard 显示存储用量
- 可选：设置最大保留天数（如 90 天后自动清理）

### 6.2 并发安全

- `session_id` 目录创建使用 `exist_ok=True`，不会冲突
- turn_number 在 append 前由数据库查询决定，避免覆盖

### 6.3 迁移兼容

- 新增 `audio_path` 列默认值为空字符串，旧数据不受影响
- 旧消息的 `audio_url` 返回 `null`，前端据此隐藏播放按钮
- 使用 `ALTER TABLE ... ADD COLUMN` 配合 `try/except` 实现幂等迁移

### 6.4 CORS / 缓存

- 音频文件与 API 同域 (`/api/audio/...`)，无跨域问题
- 建议设置 `Cache-Control: immutable`（录音内容不会变）

### 6.5 前端内存

- 使用 `URL.createObjectURL` / `new Audio(url)` 播放
- `onended` 事件中不调用 `revokeObjectURL`（因为 API URL 可复用）
- 页面卸载时无需特殊清理

## 九、不做的范围

- ❌ 不在浏览器端持久化音频（IndexedDB / localStorage）— 全部存服务端
- ❌ 不支持音频转码 — 保持 WAV 格式
- ❌ 不在前端做音频裁剪/编辑
- ❌ 不支持云端存储（S3 等）— 本地文件系统即可满足 MVP
- ❌ 不等录音上传完再转写 — 保持当前同步处理流程
